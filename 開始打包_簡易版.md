# 🚀 打包成 Windows .exe - 超簡易版

## 只需要 3 個步驟！

---

## 步驟 1️⃣：建立 GitHub Repository

1. **登入 GitHub**  
   前往：https://github.com

2. **建立新 repository**
   - 點擊右上角綠色按鈕 `New`
   - **Repository name**: `text-to-ppt-converter`
   - **選擇 Public** ✅
   - **不要勾選** "Add a README file" ❌
   - 點擊 `Create repository`

3. **記下這個網址**（待會需要）
   ```
   https://github.com/您的帳號/text-to-ppt-converter.git
   ```

---

## 步驟 2️⃣：在 Mac 上執行自動設定腳本

打開「終端機」（Terminal），執行以下指令：

```bash
# 進入專案資料夾（假設在桌面）
cd ~/Desktop/您的專案資料夾名稱

# 執行自動設定腳本
bash 快速開始.sh
```

腳本會詢問您：
- GitHub 帳號名稱
- Repository 名稱（直接按 Enter 使用預設）
- 您的名字和 Email

然後會自動：
- ✅ 設定 Git
- ✅ 加入所有檔案
- ✅ 建立 commit
- ✅ 推送到 GitHub

**如果推送時要求輸入密碼**：
- 帳號：您的 GitHub 帳號
- 密碼：需要使用 **Personal Access Token**（不是 GitHub 密碼）

### 如何產生 Token：
1. 前往：https://github.com/settings/tokens
2. 點擊 `Generate new token` → `Generate new token (classic)`
3. 勾選 `repo` 權限
4. 點擊最下方 `Generate token`
5. **複製 token**（只會顯示一次！）
6. 回到終端機，貼上 token 作為密碼

---

## 步驟 3️⃣：等待打包完成並下載

1. **前往 GitHub repository**
   ```
   https://github.com/您的帳號/text-to-ppt-converter
   ```

2. **點擊 "Actions" 標籤**
   - 會看到一個正在執行的工作流程（黃色圓圈 🟡）
   - 名稱：`Build Windows Executable`

3. **等待 3-5 分鐘**
   - 完成後會變成綠色勾勾 ✅

4. **下載 .exe 檔案**
   - 點擊完成的工作流程
   - 往下捲動到 "Artifacts" 區域
   - 點擊 `Windows-GUI-Executable` 下載
   - 解壓縮 zip 檔案
   - 就會看到 `文字轉PPT工具.exe` 🎉

---

## 完成！🎉

現在您可以：
- 將 `文字轉PPT工具.exe` 分發給 Windows 用戶
- 提供 `Windows用戶使用說明.txt`
- Windows 用戶雙擊即可使用，**無需安裝任何軟體**！

---

## 之後如何更新？

當您修改程式碼後：

```bash
# 1. 在專案資料夾中
cd ~/Desktop/您的專案資料夾

# 2. 加入變更
git add .
git commit -m "更新功能"

# 3. 推送（自動觸發打包）
git push

# 4. 前往 GitHub Actions 下載新的 .exe
```

## 疑難排解

### ❌ 推送失敗："Permission denied"

**解決**：需要設定 SSH 或使用 Personal Access Token

**快速解決（使用 Token）**：
1. 產生 Token：https://github.com/settings/tokens
2. 勾選 `repo` 權限
3. 複製 token
4. 重新推送：
   ```bash
   git push
   # 帳號：您的 GitHub 帳號
   # 密碼：貼上 token
   ```

### ❌ GitHub Actions 失敗（紅色叉叉）

**解決**：
1. 點擊失敗的工作流程
2. 查看錯誤訊息（紅色文字）
3. 通常是檔案缺失或程式碼有錯誤
4. 修正後重新推送

### ❌ 找不到 Artifacts

**原因**：工作流程還在執行中或已失敗

**解決**：
- 確認工作流程已完成（綠色勾勾）
- 如果失敗，修正錯誤後重新推送
